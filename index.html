1. Technical Documentation

Architecture & Data Model

ParcelSync is engineered on the Google AppSheet platform, utilizing Google Sheets as a high-availability relational database.

Primary Data Tables

  • Admins: Property Management oversight
  • Buildings: Master Property Registry
  • Tenants: Occupant profiles and contact info
  • Delivery Logs: Real-time transaction engine
  • Security Guards: Staff shift and deployment

Automated Logic (Bots)

  • QR Code Instant Generation
  • Push Notification on Arrival
  • Weekly PDF Management Report
  • 30-Day Automated Data Archiving

2. End-User Documentation

A Receiving a Parcel (Guard Workflow)

01

Verify if the parcel is pre-paid and non-perishable (No food items).

02

Open the Add Log form and input Courier, Tenant, and Size details.

03

Capture a clear Photo of the parcel label for record accuracy.

04

Assign the Shelf Location and click Save to trigger notifications.

B The Claim Process (Resident & Guard)

01

Resident presents their digital QR Pass and a valid government ID.

02

Guard uses the Scanner Mode to instantly retrieve the matching log.

03

Verify the Claimer Identity (Tenant or Pre-registered Helper).

04

Finalize release. Owner receives a Claim Confirmation alert instantly.

MVP Phase System Limitations

✓

Payment Processing: Excluded. Subscription billing is handled via manual B2B invoice.

✓

Hardware: No integration with physical smart lockers or turnstiles in MVP.

✓

AI Features: ID verification and OCR for labels are manual by security staff.

✓

Reminders: Schedule is fixed at Day 3 all properties.
